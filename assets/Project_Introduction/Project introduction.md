# 1. 프로젝트 개요

- **프로젝트 명**: 산업 현장 적용을 위한 비전 AI 모델의 자율 운영 플랫폼 개발
- **프로젝트 기획 배경**
    
    ![image.png](%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8%20%EC%86%8C%EA%B0%9C/image.png)
    
    - 중기부 3차년도 「AI 기반 이미지 결함탐지 시스템 개발 및 실증」과 연계
    - 실제 기업 현장에 비전 AI 모델이 배포되면, 초기 모델 성능 확보뿐 아니라 운영 단계에서의 성능 유지와 운영 리스크 대응을 위한 체계적인 관리 체계가 중요하다.
    - 기존 MLOps는 데이터·모델·배포 파이프라인의 자동화에 중점을 두어 왔으나, 실제 기업이 요구하는 수준은 이를 넘어선다. 즉, 운영 중인 모델의 상태를 실시간으로 점검하고, 성능 저하 원인을 분석하며, 재학습·검증·배포 여부를 자율적으로 판단할 수 있는 운영 관리형 AI 에이전트가 필요하다.
    - 개발된 비전 AI 모델의 실제 배포 환경을 가정하여, 추론 모니터링, 재학습 의사결정, 데이터셋 및 모델 버전 관리를 통합 지원하는 자율 운영·검증 플랫폼 개발을 목표로 한다.

# 2. 문제 정의

현재 비전 AI 운영 환경에서는 인프라 장애 대응, 예측 결과 해석, 재학습 판단이 담당자 수작업에 의존하고 있다. 이는 기존 MLOps가 학습·배포 자동화 중심으로 설계되어 운영 단계의 상태 진단, 원인 추론, 기준 기반 판정 보조, 후속 조치 연계까지 지원하지 못하기 때문이다. 그 결과 분산된 지표와 결과를 직접 해석해야 하는 비효율이 발생한다. 따라서 각 운영 기능을 Agent 기반으로 통합 고도화하여, 운영 상황을 자율적으로 감지·해석·판단·연계할 수 있는 비전 AI 자율 운영 체계를 구축하는 것을 목표로 한다.

### AS-IS/TO-BE

| Process | AS-IS | TO-BE |
| --- | --- | --- |

(DB, API) | 서비스별 개별 모니터링, 장애 대응 수작업 | Agent 기반 통합 헬스체크, 원인 분석, 자동 복구, 알림 기능 |
| 데이터 검증 및 버전 관리 | 손상·중복 데이터 유입 가능, 데이터셋 버전 관리 어려움 | 데이터 자동 검증·라우팅, 버전 관리 및 모델-데이터 추적 |
| 예측 결과 해석 및 판정 보조 | 모델 출력만으로 최종 판정 근거 부족 | 기준 문서·사내 규정 기반 설명 가능한 판정 지원 |
| 모델 상태 진단 및 재학습 판단 | 임계값 기반 단순 모니터링, 재학습 판단 수작업 | 복합 지표 기반 상태 진단, 재학습·롤백 자동 연계 |

# 3. 시스템 구성 및 Agent 설계

## 1. 인프라 관리 Agent - 통신 외부 API 정보

### 현재 문제점

- MLops 환경은 MLflow, Airflow, 모델 서빙 API, DB 등 여러 서비스가 함께 동작하는 구조이므로, 하나의 서비스만 중단되어도 전체 파이프라인이 멈추거나 일부 기능이 마비될 수 있다.
- 또한 장애 발생 시 운영자가 직접 상태를 확인하고 원인을 파악해야 하므로 대응 시간이 길어질 수 있다.

### 목적

- MLops 환경을 구성하는 여러 서비스를 자동으로 감시하고, 장애 발생 시 원인을 분석하고 복구하며, 전체 환경을 일괄적으로 제어할 수 있는 운영 보조 체계 구축

### 기능

1. **통합 헬스체크 및 자동 복구**: Agent가 주기적으로 모든 서비스의 상태를 점검하고, 서비스 다운 감지 시 의존성 순서를 고려하여 자동 재시작한다.
2. **장애 원인 분석**: 서비스 실패 시 로그를 자동 수집/분석하여 “포트 충돌”, “메모리 부족”, “디스크 용량 초과” 등 원인을 분류하고, 원인에 맞는 복구 전략을 자동 선택한다. 또한 복구 실패 시 현재 문제점과 수동 조치 방법까지 함께 제시한다.
3. **통합 알림 시스템**: 장애 발생, 서비스 복구, 성능 저하 등 이벤트를 자연어로 요약하여 Slack/이메일 전송한다. 단순 에러 로그 전달이 아닌, “MLflow Serving이 OOM으로 중단되었고, 자동 재시작하여 복구완료.”같은 상황 요약을 제공

## 2. 데이터 관리 Agent

### 현재 문제점

- 실시간으로 유입되는 이미지 데이터 중에 손상 파일, 중복 데이터 등 포함될 수 있으며, 이러한 데이터가 그대로 들어오면 학습 또는 추론 파이프라인 실패로 이어질 수 있다.
- 운영 중 데이터가 지속적으로 추가·변경되기 때문에, 어떤 데이터셋이 어떤 모델 학습 및 추론에 사용되었는지 추적하기 어렵다.

![image.png](%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8%20%EC%86%8C%EA%B0%9C/image%203.png)

### 목적

- 실시간으로 유입되는 이미지 데이터를 검증하고, 검증을 통과한 데이터만 학습 데이터셋 or 추론 데이터셋으로 활용할 수 있도록 관리한다. 또한 데이터셋의 변경 이력을 버전 단위로 관리하여, 모델과 데이터 간의 연결 관계를 추적하고 재현 가능한 운영 환경을 구축한다.

### 기능

1. 데이터 기본 검증 자동화: 파일 손상 여부, 채널 수, 중복 여부 등자동 검증하여 이상 데이터를 사전에 차단한다.
2. 데이터 라우팅 및 분류: 검증 통과 데이터는 목적에 따라 자동 분류하고, 각 저장소 및 후속 파이프라인과 연계하여 별도의 수작업 없이 학습 또는 추론 과정으로 자동 반영되도록 관리.
3. 버전관리 시스템: 검증 완료된 데이터셋을 버전 단위로 고정하고, 버전별 포함 파일, 생성 시점, 데이터 분포, 이전 버전 대비 변경 내역을 기록한다. 이를 통해 어떤 데이터가 어떤 모델 학습과 추론에 사용되었는지 추적 가능하게 하며, 필요 시 이전 버전으로 롤백하거나 동일 조건으로 재학습할 수 있도록 지원한다.
    - 데이터 버전관리 도구로는 DVC+git, lakeFS.. 등 적용

## 3. 예측 결과 사후 보정 에이전트

### 현재 문제점

- Anomaly score와 Heatmap과 같은 추론 결과는 딥러닝 모델이 생성한 출력 값이기 때문에, 현업에서 바로 최종 판정 근거로 사용하기에는 신뢰성이 충분하지 않을 수 있다.
    - 결과가 참고자료는 될 수 있어도, 최종 품질 판정 근거로 부족함

### 목적

- 추론 Output으로 나오는 Heatmap, Bounding box 이미지와 용접 품질 판정 표준(ISO 등) 도메인 지식, 사내 품질 규정, 고객사별 검사 기준을 결합하여 품질 검사 연구원의 판정 업무를 보조한다.

### 기능

1. 표준/메뉴얼 RAG 구축: 용접 관련 표준 문서, 사내 품질 매뉴얼, 작업 절차서 등을 벡터DB에 인덱싱하고, 결함 판정 시 Agent가 관련 조항을 자동으로 검색하여 제시함
    - 예: 기공 직경 2mm, ISO 기준 B등급 허용 범위 내. 단, 고객사 X의 특별 기준 적용 시 불합격
2. 판정 기준 자동 적용: 제품별, 고객사별로 다른 검수 기준을 Agent가 관리하고, 추론 결과에 해당 기준을 자동 적용하여 연구원에게 제시
3. 설명 가능한 판정 보조: 단순히 불량 또는 정상을 제시하는 것이 아니라, “모델이 해당 영역을 이상으로 탐지했으며, 관련 기준 문서의 어느 조항과 연결되는지”를 함께 설명하여 연구원의 판단을 돕는다.

### 고려해야 할 점

- 모델이 실제로 이미지 결과(Heatmap, Bounding Box)와 문서 기준을 연결해 해석할 수 있는지 검증이 필요하다.

## 4. 모델 성능 모니터링 Agent

### 현재 문제점

- 기존 MLOps 모니터링은 개별 지표의 임계값 초과 여부만 기준으로 판단하기 때문에, 이상이 감지된 이후 그 원인을 추론하는 과정은 여전히 운영자의 수작업에 의존하는 경우가 많다.
- 또한 순간적인 값만 평가하는 방식은 지속적으로 악화되는 추세와 일시적인 변동을 구분하기 어려워, 실제 대응이 필요한 상황을 놓치거나 불필요한 대응이 발생할 수 있다.

### 목적

운영 지표를 종합적으로 해석하여 모델 성능 저하 여부와 그 원인을 추론하고, 재학습 필요성을 근거와 함께 판단, 실행 할 수 있는 자율형 진단 체계를 구축한다. 이를 통해 단순 모니터링을 넘어, 운영 중인 모델의 상태를 스스로 해석하고 필요한 조치를 연결하는 운영 관리형 AI Agent를 구현하는 것을 목표로 한다.

### 기능

1. 데이터 분포 및 예측결과 통합 수집
    - 이미지 통계량과 drift 지표, 추론 결과 지표를 통합 수집
2. 복합 지표 기반 상태 판단
    - 여러 지표의 변화 패턴, 시간적 추세, 영향 범위, 과거 유사 사례를 종합적으로 분석하여 모델 상태를 진단한다. 이를 통해 단순 임계값 초과 여부를 넘어 지속적 악화와 일시적 변동을 구분하고, 관찰 / 경고 / 재학습 / 긴급 롤백의 4단계로 차등 대응한다. 또한 진단 결과에 대한 추정 원인을 자연어 근거와 함께 제시하여 운영자의 의사결정을 지원한다.
